**Workflow Explanation:**
1. **Fetch:** `RunMainLoop` pulls a batch of up to 50 messages from Redis
2. **Pre-process:** Passes the batch to `processMessages` which extracts each message and calls `parseFields` to map the raw data into standard structs
3. **Categorize:** Any malformed messages (parsing errors) are tossed into the `dead_letter_stream` for debugging. The valid ones are collected into the `validEntries` array
4. **Bulk Insert:** Calls `bulkInsert` to dump the entire array of clean data directly into the database using a single ultra-fast `COPY` command
5. **Acknowledge:** If the database insertion is successful, it reports back to Redis using the `XAck` command, allowing Redis to safely delete those logs from the Pending queue
6. **Failsafe Recovery:** If the database insert fails or the server suddenly crashes, the `XAck` command will never be called. In this case, `RunRecoveryLoop` acts as a failsafe mechanism, automatically claiming those stuck logs and rerouting them back through the pipeline starting from step 2
